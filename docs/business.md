## プロダクトビジョン

**「誰でも・どこでも・AI の助けを借りて語学を継続できる」**

語学学習の最大の課題は「継続」。LinguaCard は AI による摩擦ゼロのカード作成と科学的な SM-2 間隔反復で、継続的な語学習慣を作る。

---

## ターゲット市場

### プライマリターゲット
- **年齢層**: 20〜35歳
- **属性**: 英語・スペイン語を仕事や留学・旅行のために学ぶ社会人・大学生
- **ペイン**: 単語帳を作る手間が大きく続かない / Anki は難しすぎる
- **ゲイン**: スキマ時間（通勤・休憩）で効率的に復習できる

### セカンダリターゲット
- 英語教師・スペイン語教師（生徒向けデッキ作成・共有）
- TOEIC/TOEFL 受験者（頻出単語の効率的な暗記）

## ビジネスモデル（案）

### 現在: 完全無料（MVP 検証フェーズ）

### フェーズ2: Freemium モデル

| プラン | 価格 | 機能 |
|---|---|---|
| **Free** | 無料 | デッキ2つまで / カード30枚まで / 例文生成5回/月 |
| **Pro** | ¥980/月 or ¥8,800/年 | デッキ・カード無制限 / 例文生成無制限 / 高度な統計 |
| **Team**（将来） | ¥1,500/月・人（5人〜） | 教師用ダッシュボード / クラス管理 / デッキ共有管理 |

### 収益の考え方

**API コスト構造（claude-sonnet-4-5）:**
- 例文生成1回 = 約500 input + 300 output tokens ≈ ¥0.9/回
- Free ユーザー1人/月の最大 API コスト: 5回 × ¥0.9 = **¥4.5/人**
- Pro ユーザー1人/月の平均 API コスト（60回想定）: **¥54/人**

**Pro プランの損益（Free:Pro = 20:1 の場合）:**
```
収益              ¥980
自分の API コスト  -¥54
Free ユーザー分    -¥90（20人 × ¥4.5）
インフラ按分       -¥5
─────────────────
粗利              ¥831（約85%マージン）
```

**Free プランの制限設計の意図:**
- 5回/月 = 1週間に1〜2単語 → 「試せる」が「使い続けられない」ライン
- 30枚上限 → 30枚に達したタイミングでアップグレードを促す
- 2デッキ上限 → 3つ目のデッキを作りたくなった時がアップグレード動機

**MRR ¥50,000 到達に必要な Pro ユーザー数: 約51人**（¥480 時代の 104 人から半減）

## コスト構造

### インフラコスト（月次目安）

| 項目 | 無料プラン内 | 有料になるタイミング |
|---|---|---|
| **Supabase** | 無料（500MB DB, 50,000 MAU） | MAU 50,000 超過時（$25/月〜） |
| **Vercel** | 無料（Hobby プラン） | チーム開発・高トラフィック時 |
| **Claude API** | - | 使った分だけ（例文生成ごとに課金） |

### Claude API コスト試算
- claude-sonnet-4-5: 入力 $3/M tokens, 出力 $15/M tokens
- 例文生成1回: 約 500 input tokens + 300 output tokens ≈ $0.006（約¥0.9）
- 月30回制限（Free）: 最大 ¥27/ユーザー/月
- **Free プランのユーザー1人あたり最大 API コスト: 約¥27/月**

## 差別化戦略

### 技術的差別化
1. **AI 例文生成の品質**: Claude API により単語の品詞・難易度・言語に最適化された例文を3文生成
2. **多言語音声**: Web Speech API + 地域アクセント設定（en-US/en-GB/es-MX 等）
3. **SM-2 実装**: 科学的根拠のある間隔反復アルゴリズム

### UX 差別化
1. **摩擦ゼロのカード作成**: 単語を入力するだけで AI が例文・品詞・レベルを提案
2. **シンプルな毎日の習慣**: ダッシュボード1画面で「今日やること」が分かる
3. **ストリーク**: 継続モチベーションを可視化

## リスクと対策

| リスク | 影響度 | 発生確率 | 対策 |
|---|---|---|---|
| Claude API コスト高騰 | 高 | 低 | 生成回数制限・キャッシュ実装 |
| Supabase 利用規約変更 | 中 | 低 | DB 抽象化層の整備 |
| 競合（Anki AI 対応等） | 高 | 中 | UX の磨き込みと速度で対応 |
| ユーザーリテンション低い | 高 | 中 | 通知機能・ゲーミフィケーション強化 |
| API レート制限 | 低 | 低 | エラーハンドリング実装済み |
